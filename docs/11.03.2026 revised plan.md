## 1. Research Summary: Current State of the Codebase

### 1.1 Architecture Overview

CalibrateMe is a React + TypeScript single-page application built with Vite. It implements a **metacognitive calibration-aware adaptive learning system** for vocabulary and grammar practice (French speakers learning English). The app has a sidebar-based navigation with six tabs: Practice, Vocabulary, Analytics, Profile, Calibration Game, and Simulation Lab.

### 1.5 Identified Gaps and Issues

After thorough analysis, the following areas need revision:

#### A. Critical Functional Issues
1. **Session data not persisting across page reloads**: `allResponses` state in App.tsx is ephemeral -- when the user refreshes, all quiz responses are lost, so the Analytics dashboard starts empty every time.
2. **Grammar exercise confidence is hardcoded to 70**: In QuizContainer, `handleGrammarAnswer` always passes `confidence: 70` instead of letting the user rate their confidence, undermining the calibration measurement for grammar items.
3. **No session history view**: Users cannot see past sessions or track calibration improvement over time. The analytics only show data from the current browser session.

#### B. UX/Engagement Issues
4. **No onboarding or tutorial**: New users see the Practice screen with no explanation of what calibration is or why it matters.
5. **Session summary lacks actionable detail**: The "Needs Review" section shows raw item IDs instead of the actual vocabulary word/grammar question.
6. **No difficulty progression feedback**: Users don't see how the scheduler is adapting to them -- the calibration-aware scheduling is invisible.

#### C. Analytics & Insight Gaps
7. **No per-session ECE trend chart**: The analytics dashboard shows current ECE but not how it has changed over sessions, which is the key measure of calibration improvement.
8. **No confidence vs. accuracy scatter plot per item**: Would help users identify specific items where they are consistently miscalibrated.
9. **Dual-process classification not surfaced**: The Type1/Type2 classification runs in the engine but is never shown to the user.

#### D. Code Quality
10. **Duplicated ECE computation**: ECEMeter.tsx and LiveCalibrationCurve.tsx each have their own binning logic, separate from the core `scoringModule.ts`.
11. **No error boundaries**: A crash in any component takes down the entire app.

## 2. Revision Plan

### Priority 1: Session Persistence (Critical)
**Goal**: Analytics data survives page reloads.

- **What**: Load `allResponses` from IndexedDB on mount, save after each session.
- **Files**: `App.tsx`, `features/user/services/storageService.ts`
- **Approach**: The storageService already has IndexedDB infrastructure. Add a `loadAllResponses()` function and call it on App mount. Convert stored records back to `QuizResponse[]`.

### Priority 2: Session History & ECE Trend (Analytics)
**Goal**: Show calibration improvement over time.

- **What**: Add a new `SessionHistory` component that shows a list of past sessions with their ECE, accuracy, and calibration direction. Include an ECE-over-sessions line chart.
- **Files**: New component `features/analytics/components/SessionHistory.tsx`, update `CalibrationDashboard.tsx`
- **Data**: Each session already stores `sessionECE` via `saveSessionToProvider`. Surface this in the analytics tab.

### Priority 3: Resolve Grammar Confidence Issue
**Goal**: Grammar exercises should collect real confidence ratings.

- **What**: Add a confidence rating step to grammar exercises between answering and feedback, matching the vocabulary flow.
- **Files**: `QuizContainer.tsx`, `GrammarExercise.tsx`
- **Approach**: After the user selects a grammar option, transition to a `rate-confidence` phase before revealing correctness.

### Priority 4: Enrich Session Summary
**Goal**: Session summary shows actual item content, not raw IDs.

- **What**: Pass the quiz items into SessionSummary so "Needs Review" shows the actual word/question.
- **Files**: `SessionSummary.tsx`, `QuizContainer.tsx`
- **Approach**: Add an `items` prop to SessionSummary. Look up item content by ID for display.

### Priority 5: Surface Dual-Process Insights
**Goal**: Show users which responses were automatic vs. deliberate.

- **What**: Add a response-type breakdown to the session summary and analytics. Show "X automatic, Y deliberate" with a brief explanation.
- **Files**: `SessionSummary.tsx`, `CalibrationDashboard.tsx`
- **Data**: The QuizService already computes `ProcessedResponse` with `response_type`. Thread this through to the UI.

### Priority 6: Onboarding Card
**Goal**: First-time users understand what calibration is and how to use the app.

- **What**: Show an introductory card on first visit explaining calibration, the confidence slider, and how to interpret feedback.
- **Files**: New component `features/quiz/components/OnboardingCard.tsx`, update `QuizContainer.tsx`
- **Approach**: Check localStorage flag `calibrateme_onboarded`. Show card if not set.

### Priority 7: Error Boundary
**Goal**: Prevent full-app crashes.

- **What**: Add a React error boundary wrapper around the main content area.
- **Files**: New component `components/ErrorBoundary.tsx`, update `App.tsx`
