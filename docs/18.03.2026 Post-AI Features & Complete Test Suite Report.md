### Commit 2: ReflectivePrompt Grammar Fix (666136e)

**Bug:** The `ReflectivePrompt` modal â€” which asks learners to reflect when they are confidently wrong (confidence >= 75% and incorrect) â€” was only triggered in the vocabulary flow (`handleAnswer`). The grammar flow (`handleGrammarConfidenceSubmit`) lacked the same check, so grammar exercises never showed the reflective prompt for overconfident wrong answers.

**Fix:** Added 13 lines to `QuizContainer.tsx` in the `handleGrammarConfidenceSubmit` callback:

```typescript
// Show reflective prompt for confident misses (confidence >= 75 and wrong)
if (conf >= 75 && !correct) {
  pendingAdvanceRef.current = () => {
    setFeedbackResult(null);
    setConfidence(50);
    setGrammarPendingOption(null);
    setLastGrammarAnswer(null);
    nextItem();
  };
  setShowReflectivePrompt(true);
}
```

**Impact:** Grammar exercises now correctly trigger the metacognitive reflection prompt when the learner is confidently wrong, matching the vocabulary flow behavior. This is critical for the calibration training goal â€” overconfident grammar mistakes were silently passing without intervention.

### Mock Infrastructure

The test suite uses a single dedicated mock file:

| File | Lines | Purpose |
|------|-------|---------|
| `tests/advancedAnalyticsStore.mock.ts` | 253 | Mocks for simulationEngine, ablationRunner, sensitivityAnalysis, deltaSweep modules; provides deterministic return values for async store operations |

Other test files use inline Jest mocks (`jest.fn()`, `jest.mock()`) or test against real implementations with controlled inputs.

### Testing Patterns

1. **Direct unit testing** â€” Most test files import functions directly and test with controlled inputs (e.g., `beliefUpdateEngine`, `forgettingModel`, `statistics`)
2. **Store testing** â€” Zustand stores are tested by calling `getState()` and verifying state mutations (e.g., `simulationStore`, `advancedAnalyticsStore`)
3. **Statistical validation** â€” Tests verify numerical properties (bounds, monotonicity, convergence) rather than exact values, accounting for stochastic simulation behavior
4. **Regression testing** â€” `regression.test.ts` specifically targets previously-fixed bugs with minimal reproductions
5. **Edge case coverage** â€” Empty arrays, null values, zero-variance data, extreme profiles, and boundary conditions are systematically tested
