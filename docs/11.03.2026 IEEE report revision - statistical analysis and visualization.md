## Task 1: Ablation Study with Statistical Rigor

**Files**: `src/simulation/statisticalAnalysis.ts`, `src/simulation/ablationRunner.ts`

### Statistical Foundation (`statisticalAnalysis.ts`)

Provides inferential statistics primitives used across all new analyses.

| Function | Input | Output |
|----------|-------|--------|
| `computeStats(values)` | Array of numeric observations | Mean, SD, 95% CI (lower/upper), n |
| `computeEffectSize(group1, group2)` | Two numeric arrays | Cohen's d, pooled SD, interpretation |

**CI Computation**: Uses the t-distribution for small-sample inference:
```
CI = mean ± t_{α/2, n-1} × (SD / √n)
```

**Effect Size Interpretation** (Cohen's conventions):
| |d| | Interpretation |
|-----|----------------|
| < 0.20 | Negligible |
| 0.20–0.49 | Small |
| 0.50–0.79 | Medium |
| ≥ 0.80 | Large |

### Ablation Runner (`ablationRunner.ts`)

Orchestrates a full factorial ablation study: **6 conditions × N profiles × N seeds**.

**Default Conditions**:

| Condition | Description |
|-----------|-------------|
| Full CalibrateMe | All features enabled (dual-process, scaffolding, calibration-aware scheduling) |
| No Dual-Process | Disables dual-process (Type 1/Type 2) classification |
| No Scaffolding | Disables SRL scaffolding interventions |
| Calibration Only | Calibration-aware scheduling without dual-process or scaffolding |
| SM-2 Baseline | Standard SM-2 spaced repetition algorithm |
| BKT-Only | Bayesian Knowledge Tracing without calibration awareness |

**Output**: Per-(profile, condition) `StatisticalResult` for each metric (retention_7day, final_ece, brier_score, accuracy) plus `EffectSize` vs SM-2 baseline.

**Export**: `ablationToCSV()` generates a flat CSV for external analysis tools.

### Visualization: `AblationTable.tsx`

- Tabular display grouped by learner profile
- `StatCell`: renders mean ± CI with color coding (green for good, red for poor)
- `EffectCell`: renders Cohen's d with interpretation badge (negligible/small/medium/large)
- CSV export button in header

## Task 2: Parameter Sensitivity Analysis

**File**: `src/simulation/sensitivityAnalysis.ts`

### Purpose
Sweeps key model parameters to assess robustness and identify sensitivity thresholds.

### Default Parameter Sweeps

| Parameter | Symbol | Sweep Values |
|-----------|--------|-------------|
| Forgetting rate | λ | 0.03, 0.05, 0.08, 0.10, 0.15, 0.20 |
| Slip probability | p(slip) | 0.02, 0.05, 0.08, 0.10, 0.15 |
| Guess probability | p(guess) | 0.10, 0.15, 0.20, 0.25, 0.30 |
| Confidence noise SD | σ_c | 0.05, 0.10, 0.15, 0.20, 0.25 |
| Calibration bias | β* | -0.20, -0.10, -0.05, 0.00, 0.05, 0.10, 0.20, 0.30 |

### Method
For each (parameter value × profile × seed):
1. Run CalibrateMe simulation with the swept parameter
2. Run SM-2 baseline with the same parameter
3. Compute retention advantage = CalibrateMe retention − SM-2 retention
4. Aggregate across seeds using `computeStats()` for CIs

### Visualization: `SensitivityHeatmap.tsx`

- Heatmap grid: profiles on Y-axis, parameter values on X-axis
- Cell color: green (CalibrateMe advantage), gray (neutral), red (SM-2 advantage)
- Cell text: retention advantage ± CI
- Parameter selector tabs across the top

## Task 3: δ Dose-Response Study

**File**: `src/simulation/deltaSweep.ts`

### Purpose
Characterizes the relationship between scaffolding intensity (δ) and learning outcomes.

### Default δ Values
```
[0.00, 0.01, 0.02, 0.03, 0.05, 0.08, 0.10, 0.15]
```

### Metrics Collected Per Condition
- `final_ece`: Expected Calibration Error at end of simulation
- `retention_7day`: 7-day retention rate
- `time_to_mastery`: Sessions to reach mastery criterion
- `scaffold_count`: Total scaffolding interventions delivered

### Method
For each (δ × profile × seed):
1. Configure simulation with the specified scaffolding delta
2. Run full simulation
3. Extract outcome metrics
4. Aggregate with `computeStats()` across seeds

### Visualization: `DoseResponseChart.tsx`

- Recharts `LineChart` with δ on X-axis and selected metric on Y-axis
- Metric selector dropdown (ECE, retention, time-to-mastery, scaffold count)
- Toggle to group lines by calibration type (overconfident, underconfident, well-calibrated)
- Tooltip with exact values

## Task 6: Adaptive Difficulty Sequencing

**Files**: `src/scheduler/calibrationAwareScheduler.ts`, `src/simulation/simulationEngine.ts`, `src/types/index.ts`

### Concept
Adjusts item difficulty targeting based on current knowledge estimate (K̂) to optimize the zone of proximal development.

### Difficulty Targeting

| K̂ Range | Target Difficulty | Rationale |
|---------|------------------|-----------|
| < 0.4 (novice) | 0.17 (easy) | Build foundational confidence |
| 0.4–0.7 (intermediate) | 0.50 (medium) | Challenge at appropriate level |
| > 0.7 (advanced) | 0.83 (hard) | Push toward mastery |

### Implementation

**Scheduler** (`calibrationAwareScheduler.ts`):
```typescript
if (this.enableDifficultySequencing) {
  const difficultyBonus = -Math.abs(item.difficulty - targetDifficulty);
  score += this.difficultyWeight * difficultyBonus;
}
```

**Engine** (`simulationEngine.ts`):
- Passes `enable_difficulty_sequencing` from config to scheduler constructor
- Calls `scheduler.updateKHat(systemBelief.K_hat)` before each session's item selection

**Config** (`src/types/index.ts`):
- Added `enable_difficulty_sequencing: boolean` to `SimulationConfig` (defaults to `false`)

### UI: `SimulationControls.tsx`
- New checkbox: "Enable Adaptive Difficulty Sequencing"

## Task 7: Learner Classification Dashboard

**File**: `src/features/analytics/components/LearnerClassification.tsx`

### Purpose
Classifies the real user's calibration profile in real-time and provides personalized feedback.

### Features
- **β̂ Badge**: Color-coded badge showing estimated calibration bias
  - Green: |β̂| ≤ 0.08 (well-calibrated)
  - Yellow: 0.08 < |β̂| ≤ 0.18 (moderate bias)
  - Red: |β̂| > 0.18 (severe bias)
- **Trend Chart**: Recharts `LineChart` showing β̂ trajectory over sessions
- **Classification Label**: "Overconfident", "Underconfident", or "Well-Calibrated"
- **Personalized Description**: Context-specific explanation based on bias type and severity

### Integration
- Added to `CalibrationDashboard.tsx` with `betaHat` prop
- `App.tsx` passes `profile.learnerState.globalBetaHat` to CalibrationDashboard
