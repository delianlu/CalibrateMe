## Executive Summary

This report documents the implementation of a lightweight serverless backend for CalibrateMe using Vercel Functions and Groq's free LLM API (Llama 3.3 70B Versatile). Four AI-powered features were added: AI Flashcard Generation, AI Calibration Coach, Knowledge Graph Visualization, and AI Grammar Explainer. Together, these features add ~2,095 lines of new code across 11 new files and modifications to 8 existing files.

## Architecture Overview

```
Frontend (React/Vite)              Vercel Serverless Functions         Groq Cloud API
─────────────────────              ──────────────────────────         ──────────────
                                         api/
User clicks "AI Generate"  ──POST──→  generate-flashcards.ts    ──→  Llama 3.3 70B
User clicks "Get Coach"    ──POST──→  calibration-coach.ts      ──→  Llama 3.3 70B
User opens Knowledge Graph ──POST──→  generate-knowledge-graph.ts ──→ Llama 3.3 70B
User gets wrong answer     ──POST──→  explain-grammar.ts        ──→  Llama 3.3 70B
```

All endpoints follow the same pattern:
1. Receive structured data from the frontend via `POST`
2. Construct a domain-specific prompt with the learner's data
3. Call Groq's Llama 3.3 70B model with JSON-only system instructions
4. Parse and return structured JSON to the frontend

The API key (`GROQ_API_KEY`) is stored as a Vercel environment variable and is never exposed to the frontend.

## Part 1: AI Flashcard Generation

### Endpoint: `api/generate-flashcards.ts` (49 lines)

**Request shape:**
```typescript
interface FlashcardRequest {
  topic: string;           // e.g., "Business meetings", "Food"
  count: number;           // 5-20
  difficulty: 'easy' | 'medium' | 'hard';
  targetLanguage: 'english';
  sourceLanguage: 'french';
}
```

**Response shape:**
```typescript
interface GeneratedFlashcard {
  word: string;            // English word
  translation: string;     // French translation
  difficulty: number;      // 0.17, 0.50, or 0.83
  tags: string[];          // e.g., ["noun", "food", "academic"]
  exampleSentence: string; // Usage in context
  mnemonic: string;        // Memory trick (cognates, sound-alikes, imagery)
}
```

**Prompt design:**
- System role: "language learning expert specializing in French-to-English vocabulary"
- Temperature: 0.7 (moderate creativity for varied word selection)
- Includes example JSON format to ensure consistent output structure
- Difficulty maps to numeric values: easy → 0.17, medium → 0.50, hard → 0.83

### Frontend: `AIFlashcardGenerator.tsx` (235 lines)

**Location in app:** Vocabulary tab → "AI Generate" button in header → opens modal overlay

**States:**

| `loading` | Spinner + "Generating with AI..." + skeleton card grid |

**User flow:**

## Part 2: AI Calibration Coach

### Endpoint: `api/calibration-coach.ts` (62 lines)

**Request shape:**
```typescript
interface CalibrationCoachRequest {
  betaHat: number;                    // Calibration bias
  ece: number;                        // Expected Calibration Error
  accuracy: number;                   // 0-1
  totalSessions: number;
  recentTrend: 'improving' | 'declining' | 'stable';
  dualProcessRatio: number;           // Type 1 / total responses
  strengths: string[];
  weaknesses: string[];
  domainSplit?: {
    vocabBetaHat: number;
    grammarBetaHat: number;
  };
}
```

**Response shape:**
```typescript
interface CoachResponse {
  summary: string;        // 1-2 sentence headline
  diagnosis: string;      // What the data shows
  strategies: string[];   // 2-3 actionable strategies
  encouragement: string;  // Positive reinforcement
  focusArea: string;      // What to work on next session
}
```

**Prompt design:**
- System role: "metacognitive learning coach"
- Temperature: 0.6 (lower for more consistent, data-grounded advice)
- References Nelson & Narens metacognitive monitoring framework
- Calibration type classification: overconfident (β̂ > 0.1), underconfident (β̂ < -0.1), well-calibrated
- Type-specific coaching strategies (self-testing for overconfident, competence recognition for underconfident)
- Domain-specific notes when vocabulary/grammar split data is available

### Frontend: `CalibrationCoach.tsx` (202 lines)

**Location in app:** Analytics tab → bottom of CalibrationDashboard bento grid (requires ≥3 responses)

**States:**
| State | UI |
|-------|-----|

| Loading | Spinner + "Analyzing your learning patterns..." |
| Coaching loaded | Sectioned card: Summary → Diagnosis → Strategies → Encouragement → Focus Area |

**Sections displayed:**

**Actions:**

**Data computation:** The `CalibrationDashboard` computes ECE, accuracy, session count, and dual-process ratio inline and passes them as props to `CalibrationCoach`.

## Part 3: Knowledge Graph with AI-Generated Edges

### Endpoint: `api/generate-knowledge-graph.ts` (57 lines)

**Request shape:**
```typescript
interface KnowledgeGraphRequest {
  words: Array<{
    word: string;
    translation: string;
    tags: string[];
    mastery: number;     // 0-1
  }>;
}
```

**Response shape:**
```typescript
interface KnowledgeGraphResponse {
  edges: Array<{
    source: string;      // word
    target: string;      // word
    weight: number;      // 0-1 similarity
    relationship: string; // "synonym", "antonym", "same-category", etc.
  }>;
  clusters: Array<{
    name: string;        // e.g., "Food & Drink"
    words: string[];
    color: string;       // hex color from curated palette
  }>;
}
```

**Prompt design:**
- System role: "computational linguist"
- Temperature: 0.4 (lower for more consistent relationship detection)
- Limits: max 50 words, weight > 0.3, 1-5 edges per word, 4-8 clusters of 3-10 words
- Relationship types: synonym, antonym, same-category, part-of, causes, used-together, opposite-meaning, similar-meaning, same-domain
- Color palette: 8 distinct hex colors (#6366f1, #22c55e, #f59e0b, #ef4444, #8b5cf6, #ec4899, #14b8a6, #f97316)

### Frontend: `KnowledgeGraph.tsx` (409 lines)

**Location in app:** Vocabulary tab → "Knowledge Graph" button in header → opens full-screen overlay

**D3 Force-Directed Graph Implementation:**

| D3 Force | Configuration |
|----------|---------------|
| `forceLink` | Distance: 80, strength proportional to edge weight |
| `forceManyBody` | Strength: -200 (repulsion) |
| `forceCenter` | Centered on container midpoint |
| `forceCollide` | Radius: node radius + 8px padding |

**Node properties:**
| Property | Mapping |
|----------|---------|
| Size | 8px (new) to 20px (mastered), based on mastery: `8 + mastery × 12` |
| Fill color | Cluster color assigned by AI |
| Border color | Mastery state: green (≥80%), amber (≥40%), gray (<40%) |
| Label | Word text, always visible below node |

**Edge properties:**
| Property | Mapping |
|----------|---------|
| Thickness | Proportional to weight: `max(1, weight × 3)` |
| Style | Dashed for weak (weight < 0.5), solid for strong |
| Hover label | Relationship type shown via `<title>` element |

**Interactions:**
| Interaction | Behavior |
|-------------|----------|
| Hover node | Highlights node + connected edges + neighbors; dims unconnected nodes to 20% opacity |
| Drag node | Repositions node (D3 force simulation with alpha restart) |
| Zoom/Pan | Scroll to zoom (0.3x-4x), drag background to pan |
| Cluster filter | Click cluster pill to highlight/isolate that cluster; click again or "All" to reset |

**Tooltip (bottom-left corner):**
- Word + translation
- Mastery percentage
- Cluster name
- Tags

**Stats footer:** Node count, edge count, cluster count

**Auto-generation:** The graph auto-generates on mount. A "Regenerate" button allows re-running the AI analysis.

## Part 4: AI Grammar Explainer

### Endpoint: `api/explain-grammar.ts` (50 lines)

**Request shape:**
```typescript
interface GrammarExplainRequest {
  question: string;
  options: string[];
  userAnswer: string;
  correctAnswer: string;
  userConfidence: number;
}
```

**Response shape:**
```typescript
interface GrammarExplainResponse {
  explanation: string;     // 2-3 sentences explaining the rule
  rule: string;            // Grammar rule name
  tip: string;             // Mnemonic or strategy
  commonMistake: string;   // L1 interference explanation
}
```

**Prompt design:**
- System role: "English grammar teacher for French speakers"
- Temperature: 0.5
- When confidence > 70% and incorrect: additional prompt to address overconfidence gently
- Explicitly requests L1 (French) interference explanation
- Max tokens: 512 (shorter responses for inline display)

### Frontend: `GrammarExplanation.tsx` (186 lines)

**Location in app:** Inside QuizContainer, appears below grammar answer feedback when the user answers incorrectly

**Trigger conditions:**
- Phase is `feedback` or `reveal-answer`
- `feedbackResult === false` (incorrect answer)
- Item has `question`, `options`, and `answer` fields (grammar item)

**Automatic behavior:**
1. Component mounts → checks sessionStorage cache
2. If cached → displays immediately (no API call)
3. If not cached and under rate limit → calls API automatically
4. Displays with slide-down animation (Framer Motion)

**Rate limiting:**
- Maximum 10 AI explanations per session
- Counter stored in `sessionStorage` (`calibrateme_grammar_explain_count`)
- After limit: "AI explanations used for this session. Review your mistakes in the session summary."

**Caching:**
- Cache key: `calibrateme_grammar_explain_` + Base64 hash of `question|userAnswer`
- Stored in `sessionStorage` (clears on tab close)
- Prevents duplicate API calls for the same question

**UI sections:**
| Section | Icon | Description |
|---------|------|-------------|
| Rule | BookOpen | Grammar rule name (e.g., "Present Perfect vs Past Simple") |
| Explanation | Lightbulb | 2-3 sentences on why the correct answer is right |
| Why French Speakers Struggle | AlertTriangle | L1 interference explanation |
| Memory Tip | Lightbulb | Mnemonic strategy (green background accent) |
| Overconfidence note | — | Appears when confidence > 70% (amber left-border accent) |

### State Preservation

A new `lastGrammarAnswer` state was added to `QuizContainer` to preserve the user's selected option across the confidence → feedback phase transition (since `grammarPendingOption` gets cleared after grading).

## Security Considerations

| Concern | Mitigation |
|---------|------------|
| API key exposure | Stored as Vercel environment variable; never imported in frontend code |
| Prompt injection via user input | User input (topic, words) is embedded in structured prompts with clear instructions; responses are JSON-parsed |
| Rate limiting | Grammar explainer: 10 calls/session client-side; Groq free tier: 1,000 req/day server-side |
| Response validation | All API responses are JSON-parsed with try/catch; malformed responses show error state |
