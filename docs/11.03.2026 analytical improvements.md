## CS 6795 - Cognitive Science | Georgia Tech | Spring 2026

## Summary

Four new analysis modules and a unified Final Report component were implemented to provide deep, data-driven insights from simulation results. These modules address a key gap: while the system collected rich data across sessions (ECE, Brier, K*, dual-process classification, scaffolding counts), this data was only displayed as raw charts without synthesis or interpretation. The Final Report assembles all available metrics into a structured analytical view with natural-language summaries, trend detection, and actionable recommendations.

## Module 1: Longitudinal Analysis

**File**: `src/features/analytics/longitudinalAnalysis.ts`

### Purpose
Detects trends across sessions using linear regression and classifies learning velocity, calibration drift, and session quality.

### Key Analyses

| Analysis | Method | Output |
|----------|--------|--------|
| **ECE Trend** | Linear regression on per-session ECE values | Slope, direction (improving/declining/stable), magnitude, % change |
| **Brier Trend** | Linear regression on per-session Brier scores | Same as ECE trend |
| **Accuracy Trend** | Linear regression on per-session accuracy | Same format |
| **Knowledge Trend** | Linear regression on per-session mean K* | Same format |
| **Learning Velocity** | Threshold detection + phase classification | Sessions to 80% accuracy, sessions to good calibration (ECE < 0.10), knowledge gain rate, current phase |
| **Calibration Drift** | Confidence-accuracy gap regression | Bias trend, calibration trajectory, best calibration session |
| **Session Quality** | Coefficient of variation + delta detection | Accuracy consistency, regression sessions (>15pp drop), breakout sessions (>15pp spike) |

### Learning Phases
The module classifies learners into one of four phases based on final K* and recent gain rate:
- **Early Learning**: K* < 0.4
- **Rapid Growth**: K* 0.4–0.9 with positive gain rate
- **Plateau**: K* < 0.9 with gain rate < 0.005 after 5+ sessions
- **Mastered**: K* ≥ 0.9

### Natural Language Summary
Generates a one-sentence summary, e.g.: *"Knowledge is rapidly improving, calibration is getting better over time."*

### Exports
```typescript
analyzeLongitudinal(results: SimulationResults): LongitudinalReport
analyzeLongitudinal(sessions: SessionData[]): LongitudinalReport
```

## Module 2: Pattern Analysis

**File**: `src/features/analytics/patternAnalysis.ts`

### Purpose
Analyzes response patterns including confidence-bin accuracy distributions, dual-process breakdowns, and cognitive effort indicators.

### Key Analyses

| Analysis | Method | Output |
|----------|--------|--------|
| **Confidence Bin Patterns** | Groups sessions into 5 confidence bins (0–20%, 20–40%, ...) | Per-bin accuracy, mean RT, Type 1 ratio, calibration gap |
| **Dual-Process Breakdown** | Aggregates Type 1/Type 2 counts across sessions | Overall Type 1 ratio, trend over time, automatization session (first session with >50% Type 1) |
| **Effort Analysis** | RT trend detection + outlier flagging | Overall mean RT, trend (speeding-up/slowing-down/stable), high-effort sessions (RT > 1.5× mean) |

### Confidence Bin Calibration Gap
For each bin, the calibration gap is computed as `accuracy - midpoint_confidence`. Positive values indicate underconfidence (accuracy exceeds confidence), negative values indicate overconfidence. This provides a granular view of where miscalibration occurs along the confidence spectrum.

### Exports
```typescript
analyzePatterns(sessions: SessionData[]): PatternReport
```

## Module 3: Learner Profile Insights

**File**: `src/features/analytics/learnerInsights.ts`

### Purpose
Interprets estimated learner parameters (α, λ, β*), identifies strengths and weaknesses from session data, classifies the learner into an archetype, and generates personalized recommendations.

### Parameter Interpretation

| Parameter | Thresholds | Ratings |
|-----------|-----------|---------|
| **α (learning rate)** | ≥ 0.25 high, 0.15–0.25 average, < 0.15 low | "Fast learner", "Moderate learner", "Gradual learner" |
| **λ (forgetting rate)** | ≤ 0.07 low, 0.07–0.12 average, > 0.12 high | "Strong retention", "Typical retention", "Rapid forgetting" |
| **β* (calibration bias)** | \|β*\| ≤ 0.08 well-calibrated, 0.08–0.18 moderate, > 0.18 severe | "Well-calibrated", "Overconfident", "Underconfident" |

### Strength/Weakness Detection

Strengths and weaknesses are identified from observable session data:

- **Accuracy**: ≥80% = strength; <50% = weakness
- **Calibration**: Final ECE <8% = strength; >20% = weakness (with direction-specific advice)
- **Knowledge**: K* ≥0.9 = strength; <0.5 after 10+ sessions = weakness
- **Automaticity**: >60% Type 1 responses = strength
- **Retention**: λ <0.07 = strength; >0.12 = weakness

Each weakness includes a targeted recommendation, e.g., for overconfident learners: *"Practice recalling answers before assigning confidence. Ask: 'Can I explain this from memory?'"*

### Learner Archetypes

Nine archetypes based on parameter combinations:

| Archetype | Conditions |
|-----------|-----------|
| Calibrated Expert | Mastered + well-calibrated |
| Confident Expert | Mastered + overconfident |
| Fast but Overconfident | Fast learner + overconfident |
| Capable but Self-Doubting | Fast learner + underconfident |
| Quick Learner | Fast learner + neutral calibration |
| Optimistic Learner | Overconfident |
| Cautious Learner | Underconfident |
| High-Decay Learner | High forgetting rate |
| Steady Learner | Default |

### Scheduler Comparison

When baseline results (e.g., SM-2) are available, computes percentage improvement for:
- 1-Day, 7-Day, 30-Day Retention
- Time to Mastery
- Review Efficiency

### Exports
```typescript
analyzeLearnerInsights(
  results: SimulationResults,
  params: LearnerProfileParams,
  baselineResults?: SimulationResults
): LearnerInsightsReport
```

## Module 4: Scaffolding Impact Tracker

**File**: `src/features/analytics/scaffoldingImpact.ts`

### Purpose
Measures the effectiveness of self-regulated learning (SRL) scaffolding interventions by tracking scaffold delivery patterns and correlating with performance changes.

### Key Analyses

| Analysis | Method | Output |
|----------|--------|--------|
| **Delivery Stats** | Aggregation across sessions | Total scaffolds, sessions with scaffolds, rate per session |
| **Scaffold Trend** | Linear regression on per-session scaffold counts | Decreasing (good — calibration improving), increasing, or stable |
| **Post-Scaffold Improvement** | Compare accuracy of session N+1 vs session N when N had scaffolds | Mean accuracy change in percentage points |
| **ECE Impact** | Compare ECE of session N+1 vs session N when N had scaffolds | Mean ECE change |
| **Bias Reduction** | Compare first-third vs last-third confidence-accuracy gap | Gap reduction attributable to scaffolding |

### Phase Analysis

Splits the simulation into three phases (early, middle, late) and reports for each:
- Scaffolds per session
- Mean ECE
- Mean accuracy

This reveals whether scaffolding frequency decreases as the learner improves — the expected pattern if scaffolding is effective.

### Exports
```typescript
analyzeScaffoldingImpact(sessions: SessionData[]): ScaffoldingReport
```

## Final Report Component

**File**: `src/features/analytics/components/FinalReport.tsx`

### Purpose
Unified React component that synthesizes all four analysis modules into a single, structured report view with charts, tables, and natural-language insights.

### Report Sections

| # | Section | Content | Visualization |
|---|---------|---------|---------------|
| 1 | **Executive Summary** | Learner archetype, final K*, final ECE, mastery status, 30-day retention | Metric tiles with archetype badge |
| 2 | **Learning Trajectories** | K* and K̂ over sessions, ECE and Brier over sessions | Dual Recharts AreaChart + LineChart |
| 3 | **Learning Velocity** | Sessions to 80% accuracy, sessions to good calibration, gain rate, current phase | Metric card grid |
| 4 | **Response Patterns** | Confidence-bin accuracy, dual-process breakdown | BarChart + progress bar |
| 5 | **Learner Parameters** | α, λ, β* with interpretations and ratings | Parameter cards with rating badges |
| 6 | **Strengths & Weaknesses** | Identified areas with evidence and recommendations | Two-column card layout |
| 7 | **Scaffolding Impact** | Effectiveness metrics and phase table | Metric grid + data table |
| 8 | **Scheduler Comparison** | CalibrateMe vs baseline metrics | Comparison table with Δ% |
| 9 | **Session Quality** | Accuracy consistency, regression/breakout sessions | Metric cards |
| 10 | **Recommendations** | Personalized, data-driven suggestions | Styled list |

### Export Capabilities

Two export formats available via header buttons:
- **Full Report (JSON)**: Complete analytical output including all module results
- **Session Data (CSV)**: Raw per-session data for external analysis

### Integration

The FinalReport is accessible from the **Simulation Lab** tab. After running a simulation, a **"View Full Report"** button appears above the standard results. Clicking it replaces the default charts with the comprehensive report view. A **"Back to Results"** button returns to the standard view.

**Files Changed**:
- `src/components/Dashboard.tsx` — Added FinalReport import, `showReport` state, conditional rendering
- `src/App.css` — 350+ lines of new CSS for report layout, metric tiles, parameter cards, dual-process bar, tables, and responsive grid

### Barrel Export

All new modules are exported from the analytics barrel:

