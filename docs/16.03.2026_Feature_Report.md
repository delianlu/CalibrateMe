## Executive Summary

CalibrateMe implements six core features that together form a metacognitive calibration-aware adaptive learning system. The application combines cognitive science engines (BKT, calibration scoring, dual-process classification, adaptive scaffolding) with an interactive React UI to measure and improve the alignment between a learner's confidence and their actual knowledge.

**Total source code:** ~12,400 lines across 75+ TypeScript/TSX files
**Test coverage:** 14 test suites, 162 tests
**Tech stack:** React 18, TypeScript, Vite 5, Zustand, Recharts, Framer Motion

## Feature 1: Practice Mode

**Navigation tab:** Practice | **Icon:** Brain
**Source:** `src/features/quiz/` (~2,045 lines)

### Description

The Practice feature provides an interactive flashcard and grammar exercise interface where learners answer questions, rate their confidence, and receive adaptive scaffolding prompts. Response time is measured automatically to enable dual-process classification (Type 1 automatic vs. Type 2 deliberate responses).

### Key Components

| Component | File | Purpose |
|-----------|------|---------|
| `QuizContainer` | `components/QuizContainer.tsx` (494 lines) | Main orchestrator — manages quiz flow, session state, item scheduling, and scaffolding integration |
| `Flashcard` | `components/Flashcard.tsx` | Card-flip UI for vocabulary items with front/back reveal |
| `GrammarExercise` | `components/GrammarExercise.tsx` | OffGrid grammar exercise interface (329 items) |
| `ConfidenceSlider` | `components/ConfidenceSlider.tsx` | 0–100% continuous confidence rating input |
| `AnswerButtons` | `components/AnswerButtons.tsx` | Multiple-choice response selection |
| `SessionSummary` | `components/SessionSummary.tsx` | Post-session performance breakdown |
| `ReflectivePrompt` | `components/ReflectivePrompt.tsx` | Displays adaptive scaffolding prompts for miscalibrated learners |
| `OnboardingCard` | `components/OnboardingCard.tsx` | First-time user welcome and instructions |
| `QuizProgressPath` | `components/QuizProgressPath.tsx` | Visual progress indicator through session |

### Data Sources

- **Vocabulary packs** (200+ items): Essential English, Academic English, Business English (`src/data/vocabularyPacks/`)
- **Grammar exercises** (329 items): OffGrid activities adapted via `src/data/offgridAdapter.ts`

### Cognitive Science Integration

- **BKT belief update** (`src/bkt/beliefUpdateEngine.ts`) — updates knowledge posterior after each response using correctness, confidence, and response time as evidence channels
- **Calibration-aware scheduler** (`src/scheduler/calibrationAwareScheduler.ts`) — selects next item based on BKT posterior, calibration bias (beta-hat), and coverage
- **Dual-process classifier** (`src/dualProcess/classifier.ts`) — classifies each response as Type 1 or Type 2 via z-score normalized response time
- **Adaptive scaffolding** (`src/scaffolding/adaptiveScaffolding.ts`) — triggers reflection prompts for overconfident learners, encouragement for underconfident

### Hooks & Services

- `useQuizSession` — manages quiz session lifecycle and state transitions
- `useResponseTimer` — high-precision response time measurement
- `quizService.ts` — item selection, scheduling, and session management logic

The Analytics feature provides a comprehensive calibration dashboard that visualizes learner performance across multiple dimensions. ### Analysis Engines

| Module | File | Capabilities |
|--------|------|-------------|
| `learnerInsights.ts` | `features/analytics/` | 8 learner archetypes (Dunning-Kruger, Imposter, Calibrated Expert, etc.), strength/weakness detection, personalized recommendations |
| `longitudinalAnalysis.ts` | `features/analytics/` | Trend detection (improving/declining/stable ECE), phase classification (mastered/early-learning/rapid-growth/plateau), learning velocity, calibration drift |
| `patternAnalysis.ts` | `features/analytics/` | Confidence bin analysis, dual-process breakdown (Type 1 ratio), automatization session detection, effort analysis (RT trends) |
| `scaffoldingImpact.ts` | `features/analytics/` | Measures pre/post scaffolding calibration changes |

### Calibration Metrics

- **ECE** (Expected Calibration Error) — mean absolute gap between confidence and accuracy across bins
- **Brier Score** — mean squared difference between confidence and binary outcome
- **MCE** (Maximum Calibration Error) — worst single-bin calibration gap
- **Beta-hat** — signed calibration bias (positive = overconfident, negative = underconfident)

### 8 Learner Archetypes

1. Dunning-Kruger Effect — high confidence, low accuracy
2. Imposter Syndrome — low confidence, high accuracy
3. Calibrated Expert — high confidence, high accuracy, well-calibrated
4. Calibrated Novice — low confidence, low accuracy, well-calibrated
5. Volatile Learner — inconsistent confidence and accuracy patterns
6. Rapid Improver — improving calibration trajectory
7. Plateau Learner — stalled calibration improvement
8. Consistent Performer — stable, moderate calibration

## Feature 3: Simulation Lab

**Navigation tab:** Sim Lab | **Icon:** FlaskConical
**Source:** `src/components/Dashboard.tsx` + `src/simulation/` + `src/store/` + `src/workers/` (~2,773 lines)

### Description

The Simulation Lab enables Monte Carlo simulations that compare CalibrateMe's calibration-aware scheduling against baseline algorithms (SM-2, BKT-Only, Decay-Based) across 15 learner profiles. It supports single-profile simulations, multi-algorithm comparisons, and hypothesis testing — all running in a background Web Worker to keep the UI responsive.

### Key Components

| Component | File | Purpose |
|-----------|------|---------|
| `Dashboard` | `components/Dashboard.tsx` (398 lines) | Main simulation interface — profile selection, algorithm comparison, hypothesis testing |
| `CalibrationChart` | `components/CalibrationChart.tsx` | Session-by-session calibration trajectory visualization |
| `CalibrationCurve` | `components/CalibrationCurve.tsx` | Calibration curve for simulated data |
| `ComparisonView` | `components/ComparisonView.tsx` | Side-by-side algorithm comparison results |
| `HypothesisResults` | `components/HypothesisResults.tsx` | Displays H1/H2/H3 hypothesis test outcomes |
| `MetricsDisplay` | `components/MetricsDisplay.tsx` | Key simulation metrics (retention, ECE, mastery time) |

### Simulation Engine

| Module | File | Purpose |
|--------|------|---------|
| `simulationEngine.ts` | `src/simulation/` | Orchestrates full experiment: profile creation, N sessions, response generation, state updates, trajectory recording |
| `responseGenerator.ts` | `src/simulation/` | Synthetic learner responses — P(correct) = K* (1-slip) + (1-K*) guess; confidence = K* + beta* + noise |
| `statisticalAnalysis.ts` | `src/simulation/` | Mean, SD, 95% CI (t-distribution), Cohen's d with pooled SD |

### Learner Profiles (15 total)

**Core 3x3 Matrix (9 profiles):**

| | Low Ability | Medium Ability | High Ability |
|---|---|---|---|
| **Overconfident** | Low-Over | Med-Over | High-Over |
| **Underconfident** | Low-Under | Med-Under | High-Under |
| **Well-Calibrated** | Low-Well | Med-Well | High-Well |

**Extended Boundary Cases (6 profiles):** Near-Zero, Near-Ceiling, Extreme-Over, Extreme-Under, Fast-Forgetter, Slow-Learner, plus the Crammer profile.

### Scheduling Algorithms Compared

1. **CalibrateMe** — full calibration-aware scheduling with dual-process bonus and scaffolding
2. **SM-2** — SuperMemo 2 baseline (easiness factor, graduated intervals)
3. **BKT-Only** — Bayesian Knowledge Tracing without calibration adjustment
4. **Decay-Based** — pure exponential decay scheduling

### Infrastructure

- **Web Worker** (`src/workers/simulationWorker.ts`) — runs simulations off the main thread; throttles progress messages to 50ms
- **Zustand Store** (`src/store/simulationStore.ts`) — manages simulation state, comparison results, and hypothesis test outcomes
- **React Hook** (`src/workers/useSimulationWorker.ts`) — wraps worker lifecycle with `runAblation`, `runSensitivity`, `runDeltaSweep`, `cancel`

### Hypothesis Testing

| Hypothesis | Claim | Metric |
|------------|-------|--------|
| **H1** | Overconfident learners show the largest retention improvement under CalibrateMe | 7-day retention vs SM-2 |
| **H2** | Underconfident learners show moderate improvement | 7-day retention vs SM-2 |
| **H3** | Well-calibrated learners show minimal difference (validates calibration as key variable) | 7-day retention vs SM-2 |

## Feature 4: Gamification

**Source:** `src/features/gamification/` (~785 lines)

The Gamification feature provides extrinsic motivation through an XP/leveling system, streak tracking, achievement badges, and calibration-specific bonuses. It rewards not just correctness but also calibration improvement, encouraging learners to develop accurate self-assessment.

|-----------|------|---------|

| `NotificationToast` | `components/NotificationToast.tsx` | Toast notifications for XP gains, achievements, and streaks |

### Engine & Logic

| Module | File | Purpose |
|--------|------|---------|
| `gamificationEngine.ts` | `features/gamification/` | Core engine: XP calculation, level thresholds, streak logic, session processing |
| `achievements.ts` | `features/gamification/` | Achievement definitions, unlock conditions, and progress tracking |
| `types.ts` | `features/gamification/` | GamificationState type, achievement categories, notification types |

### XP System

- **Base XP per correct answer** — scaled by difficulty
- **Calibration bonus** — extra XP when confidence closely matches accuracy (rewarding good calibration)
- **Streak multiplier** — consecutive correct answers increase XP gain
- **Level progression** — exponential XP curve for leveling up

### Achievement Categories

Achievements span multiple dimensions:
- Practice milestones (sessions completed, items mastered)
- Calibration milestones (ECE thresholds, calibration streaks)
- Streak milestones (consecutive days, consecutive correct)
- Exploration milestones (features used, vocabulary packs completed)

### Integration

The gamification engine hooks into the practice flow via `processSession()` in `App.tsx`, which is called after each quiz session completes. Notifications are managed at the app level and displayed via `NotificationToast`. Confetti is triggered on significant events (level-ups, rare achievements) using the `canvas-confetti` library.

## Feature 5: Cal Game (Calibration Mini-Game)

**Navigation tab:** Cal Game | **Icon:** Gamepad2
**Source:** `src/features/minigame/` (~945 lines)

### Description

The Cal Game is a standalone 10-question calibration training trivia game. Players answer general-knowledge questions and rate their confidence before seeing the result. The game specifically trains metacognitive calibration by providing immediate feedback on the gap between confidence and accuracy, helping learners internalize what different confidence levels feel like.

### Key Components

| Component | File | Purpose |
|-----------|------|---------|
| `MiniGameContainer` | `components/MiniGameContainer.tsx` (332 lines) | Full game flow: question display, confidence rating, answer reveal, scoring, results screen |
| `MiniGameResults` | `components/MiniGameResults.tsx` | Post-game results with calibration curve, accuracy percentage, and calibration score |

### Game Flow

1. **Question phase** — display trivia question with 4 multiple-choice options
2. **Confidence phase** — player rates confidence (0–100%) before seeing result
3. **Reveal phase** — answer revealed with immediate calibration feedback
4. **Results phase** — after 10 questions, show calibration curve, accuracy, Brier score, and per-question breakdown

### Question Bank

- **Source:** `questions.ts` — curated set of general-knowledge trivia questions
- **Categories:** science, history, geography, mathematics, language
- **Format:** question text, 4 options, correct answer index, difficulty rating

### Scoring Mechanism

- **Accuracy score** — percentage of correct answers
- **Calibration score** — based on Brier score (squared difference between confidence and binary outcome)
- **Combined feedback** — shows both accuracy and calibration quality, with specific advice for improving metacognitive awareness

### Implementation Details

- **Lazy-loaded** via `React.lazy()` to reduce initial bundle size
- **Framer Motion animations** for question transitions and reveal effects
- **Chart visualization** (Recharts) for post-game calibration curve comparing player's confidence-accuracy pairs against the perfect calibration diagonal
- **Standalone state** — independent from the main practice system, enabling quick calibration training sessions

## Feature 6: Advanced Analytics

**Navigation tab:** Sim Lab (Advanced tab) | **Icon:** FlaskConical
**Source:** `src/features/simulation/` + `src/simulation/` (ablation, sensitivity, delta sweep) (~2,700+ lines combined)

### Description

The Advanced Analytics feature provides research-grade statistical analysis tools: multi-seed ablation studies, parameter sensitivity analysis, and scaffolding dose-response (delta) sweeps. All analyses produce publication-quality visualizations with 95% confidence intervals and Cohen's d effect sizes, suitable for inclusion in IEEE-format research papers.

### Key Components

| Component | File | Purpose |
|-----------|------|---------|
| `AblationTable` | `components/AblationTable.tsx` | Tabular display of ablation results across all conditions and profiles with effect sizes |
| `SensitivityHeatmap` | `components/SensitivityHeatmap.tsx` | Heatmap visualization of retention advantage across parameter sweeps |
| `DoseResponseChart` | `components/DoseResponseChart.tsx` | Delta dose-response curves with 95% CI bands |
| `MasteryComparison` | `components/MasteryComparison.tsx` | Time-to-mastery comparison across schedulers |
| `ScheduleComparison` | `components/ScheduleComparison.tsx` | Side-by-side scheduling algorithm comparison |

### Analysis Modules

| Module | File | Simulations | Purpose |
|--------|------|-------------|---------|
| `ablationRunner.ts` | `src/simulation/` | 9 profiles x 6 conditions x 30 seeds = 1,620 (core) + 1,080 (extended) | Ablation study removing one component at a time |
| `sensitivityAnalysis.ts` | `src/simulation/` | 3 profiles x 5 params x 5–8 values x 10 seeds = 1,340 | Parameter sensitivity sweeps (lambda, beta, slip, guess, noise) |
| `deltaSweep.ts` | `src/simulation/` | 9 profiles x 8 delta values x 15 seeds = 1,080 | Scaffolding rate dose-response analysis |

**Total: 5,160 Monte Carlo simulations** in a single analysis run (~86 seconds).

### Ablation Conditions (6)

1. **Full CalibrateMe** — all components active
2. **No Dual-Process** — remove Type 1/Type 2 classification
3. **No Scaffolding** — remove adaptive scaffolding prompts
4. **Calibration Only** — calibration adjustment without dual-process or scaffolding
5. **SM-2 Baseline** — standard SuperMemo 2 algorithm
6. **BKT-Only** — Bayesian Knowledge Tracing without calibration

### Sensitivity Sweeps (5 parameters)

| Parameter | Symbol | Sweep Range | Effect |
|-----------|--------|-------------|--------|
| Forgetting rate | lambda | 0.01–0.15 | How quickly knowledge decays |
| Calibration bias | beta* | -0.3–+0.3 | Over/underconfidence magnitude |
| Slip rate | slip | 0.01–0.20 | P(incorrect given knowledge) |
| Guess rate | guess | 0.05–0.40 | P(correct given no knowledge) |
| Response noise | noise | 0.1–0.5 | Variability in response times |

### Statistical Methods

- **95% Confidence intervals** via t-distribution (df = n-1)
- **Cohen's d effect sizes** with pooled standard deviation (thresholds: small 0.2, medium 0.5, large 0.8)
- **Paired comparisons** against SM-2 baseline for retention advantage
- **Multi-seed aggregation** for robust estimates (10–30 seeds per condition)

### State Management

- **Zustand store** (`src/store/advancedAnalyticsStore.ts`) — manages ablation, sensitivity, and delta sweep state with progress tracking
- **Web Worker delegation** — heavy simulations run in background thread via `simulationWorker.ts`
- **Fallback** — main-thread `setTimeout` chunking for Node.js/test environments

### Output Formats

- **Interactive charts** in the browser (Recharts with ExportableChart wrapper)
- **CSV export** for external analysis (8 CSV files)
- **LaTeX tables** for IEEE paper inclusion (6 .tex files with booktabs formatting)
- **PNG/SVG export** for all chart visualizations
